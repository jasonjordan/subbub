What is subbub?
subbub listens to the audio of whatever you are watching and translates it into subtitles in real time in the language you choose. You can keep watching while reading captions over any app.

Supported Devices
subbub works on Android TV and Google TV running Android 13, 14 or 15.

How to Use

1. Enable the Accessibility Service
Before you can use subbub, you must enable it in Settings - Accessibility - subbub Subtitles. This is required because subbub uses Android's accessibility overlay to display captions reliably over every app. Without it, subtitles cannot appear.

2. Choose an audio source
- System Audio captures sound directly from the TV. Best quality, no background noise.
- Microphone uses the device microphone. Can pick up room noise and is less accurate.

3. Choose the Spoken language
The language being spoken on screen (for example, English).

4. Choose the Subtitle language
The language you want subtitles to appear in (for example, Polish).

Example: Watching an English show and want Polish subtitles? Set Spoken language to English and Subtitle language to Polish.

5. Press Start Subtitles
The app will ask for the microphone permission, then subtitles will appear over whatever you are watching.

6. Stop anytime
Pull down the notification shade and tap the Stop button.

Permissions

Accessibility Service
Required. subbub uses Android's built-in accessibility system to draw captions over YouTube, live TV, and every other app. This is the only reliable way to display subtitles on Android TV. The service does not read screen content or interact with apps - it only draws text at the bottom of the screen.

Record audio
Required for Microphone mode. Also required by Android even when using System Audio mode.

Capture screen / MediaProjection
Required only for System Audio mode. Android shows a system dialog asking to allow capture. subbub uses this only for the audio stream, not video recording.

What Works

- Free-to-air TV - best case. Broadcast audio is not blocked.
- YouTube on Android TV - generally allows audio capture.
- Any app with audio - as long as the audio is not DRM-protected.

What Does Not Work

- Netflix, Disney+, Spotify - these apps block audio capture for copyright. Use Microphone mode instead, or use a second device.
- System Audio on some TVs - certain TV manufacturers disable MediaProjection audio capture. If System Audio fails, use Microphone mode.
- Cheap TV sticks - with less than 2 GB RAM may struggle.

Troubleshooting

Subtitles do not appear over other apps
Make sure the subbub Accessibility Service is turned ON in Settings - Accessibility - subbub Subtitles. This is the most common cause. If it is off, the Start button will be disabled and a red banner appears in the app.

Subtitles stop when I switch apps
If you are using Microphone mode, subbub runs in the background and should continue. Make sure the notification shows "subbub - Listening via microphone...". If it stops, your TV may be killing background services aggressively.

Fixes:
- Go to Settings - Apps - subbub - Battery and set it to Unrestricted or Don't optimise.
- Some TVs have a Background process limit in Developer Options - set it to Standard limit.

System Audio mode does not capture anything
System Audio uses Android's MediaProjection API. Some TV manufacturers (notably Xiaomi, Fire TV, and certain Sony models) disable or restrict this API.

Check:
- Did you tap Allow on the system "Start recording or casting?" dialog?
- Is the app you are watching known to block capture? (Netflix, Disney+, Spotify always block it.)
- Try Microphone mode instead - it works on every device.

Subtitles never appear
Make sure all required items are set up:
- subbub Accessibility Service is enabled in Settings
- Microphone permission is granted
- Screen capture permission was granted (for System Audio mode)

If any were denied, go to Settings - Apps - subbub - Permissions and enable them manually.

Speech and Translation Models

subbub uses the open-source Vosk speech-recognition engine for both Microphone and System Audio modes. The first time you use a spoken language, a small model (about 40-50 MB) is downloaded over Wi-Fi. Translation models are also downloaded once per language pair. After that, everything works offline.
